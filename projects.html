Here Are Some of My Projects

In Summer 2024 I joined the GearUp transfer & research program (continued beyond the term) and worked with a graduate student and Prof. Kirill Levchenko on Android-side GNSS spoofing/jamming detection that requires no extra sensors. Our core approach targeted spoofing (jamming is simpler to flag as overwhelming in-band noise) by exploiting Doppler: since real satellites move relative to a receiver, we can use public ephemerides to verify line-of-sight visibility and predict the received frequency; a terrestrial spoofer won't match the expected Doppler (or will deviate in a calculable way if the receiver is moving). I implemented the prediction pipeline in Pythonâ€”ephemeris ingest, ECEF/ECI transforms, Earth rotation, receiver kinematics, and expected-frequency computation. To ground the model, I collected labeled datasets with Google's GNSS Logger (stationary baselines, constant-speed motion, and repeated back-and-forth walks on the UIUC Quad). We also ran small jamming trials with a HackRF and investigated spoofing testbeds (a GPS signal generator that proved unsuitable, then HackRF + GNU Radio). By January 2025, I wrapped my contributions to focus on coursework: the math and calculations were working, with remaining engineering around reliably extracting satellite IDs and frequencies across multiple constellations (GPS, GLONASS, Galileo, BeiDou).

Encoded and Decoded Output

Automated validity checking output ensuring decoded output matches original.

I developed a C++ implementation of RSA encryption that securely processes arbitrary input files using user-specified key parameters (p, q, e). The program generates encrypted and decrypted outputs, automatically validating correctness by comparing results. Unlike simpler text-based implementations, this project handles all character sets - including non-ASCII symbols, foreign alphabets, and raw binary data - making it robust for diverse input types. This project shaped my ability to work with number theory, modular arithmetic, and file I/O while building a fully functional cryptosystem from the ground up.

As my first major C++ project, I built a console-based Battleship game that emphasized intuitive user interaction and robust input validation. The program supports both a debug mode with visible computer boards and a standard mode with hidden boards, letting players set up ships with simple commands for orientation and placement. I implemented safeguards against invalid input, overlapping ships, and out-of-bounds placements to ensure smooth gameplay. Once the game begins, the player and computer alternate turns, with boards updated visually after each move using symbols for hits, misses, and unguessed positions. While the computer's strategy is randomized, the player receives clear feedback on hits and when ships are sunk, creating an accessible and engaging experience.
